خطوات بسيطة لبدء التشغيل خلال 10 دقائق

1

أنشئ Repository على GitHub

Terminal
# إنشاء مجلد المشروع
mkdir football-comedy-ai && cd football-comedy-ai
# تهيئة Git repository
git init
# إنشاء هيكل الملفات
mkdir -p .github/workflows scripts videos output
2

أضف GitHub Actions Workflow

.github/workflows/main.yml 
name: 🎬 Football Comedy Auto-Upload

on:
  schedule:
    - cron: '0 */6 * * *'  # كل 6 ساعات
  workflow_dispatch:  # تشغيل يدوي

jobs:
  create-content:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Install dependencies
        run: |
          pip install requests google-auth google-auth-oauthlib google-auth-httplib2 google-api-python-client
      
      - name: Fetch Match Data
        env:
          API_FOOTBALL_KEY: ${{ secrets.API_FOOTBALL_KEY }}
        run: python scripts/fetch_matches.py
      
      - name: Generate Comedy Script
        env:
          DEEPSEEK_KEY: ${{ secrets.DEEPSEEK_KEY }}
          GEMINI_KEY: ${{ secrets.GEMINI_KEY }}
        run: python scripts/generate_script.py
      
      - name: Upload to YouTube
        env:
          YOUTUBE_CLIENT_ID: ${{ secrets.YOUTUBE_CLIENT_ID }}
          YOUTUBE_CLIENT_SECRET: ${{ secrets.YOUTUBE_CLIENT_SECRET }}
        run: python scripts/upload_youtube.py
3

إعداد GitHub Secrets

⚠️ تحذير أمني مهم!

لا تضع مفاتيح API مباشرة في الكود! استخدم GitHub Secrets لحمايتها.

# اذهب إلى: Settings → Secrets and variables → Actions
أضف المفاتيح التالية:
• YOUTUBE_CLIENT_ID
• YOUTUBE_CLIENT_SECRET
• DEEPSEEK_KEY
• GEMINI_KEY
• API_FOOTBALL_KEY

الأمان والحماية

حماية مفاتيح API والبيانات الحساسة أولوية قصوى

🚨 تنبيه أمني عاجل!

المفاتيح التي شاركتها في رسالتك مكشوفة الآن للعامة! يجب عليك اتخاذ الإجراءات التالية فوراً:

1. أعد إنشاء جميع مفاتيح API من لوحات التحكم
2. احذف المفاتيح القديمة المعرّضة
3. استخدم GitHub Secrets لتخزين المفاتيح الجديدة
4. لا تشارك المفاتيح أبداً في الكود أو المحادثات

الممارسات الآمنة

  • استخدام GitHub Secrets لجميع المفاتيح
  • تفعيل المصادقة الثنائية 2FA
  • تحديث المفاتيح دورياً كل 90 يوم
  • مراجعة صلاحيات OAuth بانتظام
  • استخدام نطاقات Redirect محددة

ما يجب تجنبه

  • وضع المفاتيح في الكود المباشر
  • مشاركة المفاتيح في المحادثات
  • استخدام نفس المفتاح في مشاريع متعددة
  • ترك Redirect URI مفتوح (*)
  • إهمال مراجعة سجلات الوصول
